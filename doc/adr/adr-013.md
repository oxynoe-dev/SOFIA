# ADR-013 — Identifiants techniques en anglais

**Statut** : Accepted
**Date** : 2026-04-16
**Auteur** : Aurele
**Review** : @solene (2026-04-16) — ratifie les 4 points ouverts

## Contexte

Les identifiants techniques du protocole H2A (marqueurs de friction, tags de resolution, directions, types de contribution, champs frontmatter) sont aujourd'hui en francais. Ils traversent 3 couches : protocol/ (spec), implementation/ (Python, JSON), dashboard (HTML/JS).

Le protocole est destine a etre publie en anglais. Traduire les identifiants plus tard implique un refactoring simultane dans le protocol, le parser, le schema JSON, le dashboard, les conventions d'instance et les sessions historiques.

## Decision

Les identifiants techniques passent en anglais. Le contenu (descriptions, documentation) reste en francais tant que le protocol n'est pas traduit.

## Points resolus (review @solene 2026-04-16)

1. **`juste` → `sound`** — terme Toulmin exact. `correct` implique verite objective, `sound` capture la solidite argumentative. Ratifie.

2. **`faux` → `refuted`** — acte argumentatif (rebuttal), pas jugement de verite. Maintient le registre argumentatif des autres marqueurs. Elimine la collision booleen. Ratifie.

3. **Retrocompatibilite** — parser bilingue (FR+EN) pendant la transition. Le mapping FR→EN doit etre documente explicitement dans le parser (pas un alias silencieux). Ratifie.

4. **`matiere` → `substance`** — apport de fond, registre philosophie de l'argumentation. `material` trop physique, `content` trop generique, `input` trop processuel. Ratifie.

## Consequences

- Les identifiants sont stables pour la publication anglaise du protocole
- Le parser accepte FR et EN pendant la transition
- Pas de reecriture de l'historique — les sessions historiques restent en francais
- Le dashboard affiche les labels EN (ou traduits selon la locale, hors scope)
