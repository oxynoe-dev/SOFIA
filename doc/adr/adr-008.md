# ADR-008 — Restructuration Core / Protocol / Runtime

**Statut** : Accepted
**Date** : 2026-04-06
**Auteur** : Mira

## Contexte

Le répertoire `core/` mélangeait deux natures de contenu : les invariants de la méthode (principes, personas, friction, devoirs) et le contrat d'interface (artefacts, conventions, traçabilité, isolation, orchestration, instance). De même, `claude-code/` vivait à la racine alors qu'il s'agit d'une implémentation remplaçable.

Cette confusion rendait la frontière floue entre ce qui est portable et ce qui ne l'est pas. En vue de la publication (v0.3) et du multi-provider (v0.5), il faut que la structure du repo reflète le modèle conceptuel.

## Décision

Restructurer `voix/` en trois couches explicites :

**Core** — Les invariants. Principes, personas, friction, devoirs. Ce qui ne change pas quand on change d'outil. Templates : persona + 7 archétypes.

**Protocol** — Le contrat d'interface. Artefacts, conventions, traçabilité, isolation, orchestration, instance. Fichiers, pas API. Git, pas base de données. C'est ce qui rend Voix portable. Templates : note, review, feature, adr, session, roadmap, team-orga, voix-instance.

**Runtime** — L'implémentation concrète. Claude Code aujourd'hui (claude-md, mémoire, sessions, hooks). Remplaçable sans toucher au core ni au protocol. Templates : claude-md, workspace scaffold.

```
voix/
├── core/              ← invariants + templates persona
├── protocol/          ← contrat d'interface + templates bus
├── runtime/
│   └── claude-code/   ← implémentation + templates runtime
└── doc/
```

## Alternatives considérées

1. **Garder la structure plate (core/ + claude-code/)** — Simple mais confond deux niveaux d'abstraction. La question "qu'est-ce qui est portable ?" n'a pas de réponse évidente dans la structure.

2. **Deux couches (method/ + implementation/)** — Ne nomme pas explicitement le protocol, qui est pourtant la couche clé de portabilité.

3. **Quatre couches (core/ + protocol/ + runtime/ + tooling/)** — Séparer templates dans tooling/. Rejeté : les templates incarnent leur couche, les séparer casse la cohérence.

## Conséquences

- Les chemins internes changent — tous les liens mis à jour (doc, workflows, CLAUDE.md, README)
- Les ADR historiques (003, 006) ne sont pas modifiés (convention : on ne corrige pas l'historique)
- Le CHANGELOG enregistre le changement comme un breaking change interne
- La structure est prête pour `runtime/mistral/` en v0.5 sans nouvelle restructuration
- Les templates sont distribués dans leur couche, pas centralisés — un contributeur sait immédiatement à quelle couche appartient un template

## Origine

Proposition de Sofia (session graphisme 2026-04-05) identifiant les trois niveaux : "Core — les invariants. Protocol — le contrat d'interface. Runtime — l'implémentation concrète." Validé et implémenté par Mira, arbitré par le PO.
