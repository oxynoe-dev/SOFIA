# ADR-006 — Isolation `core/` pré-publication

**Statut** : Accepted
**Date** : 2026-04-05
**Auteur** : Mira

## Contexte

Voix se prépare à la publication (v0.3.0). La structure actuelle mélange le cœur de la méthode (principes, personas, orchestration...) avec l'implémentation Claude Code et la documentation utilisateur. Pour accueillir des PR et des contributions, il faut une frontière claire entre ce qui est la méthode (stable, provider-agnostic) et ce qui est autour.

## Décision

Restructurer voix/ avec un répertoire `core/` qui contient uniquement le cœur de la méthode : principes, personas, friction, isolation, orchestration, traçabilité, artefacts, instance, devoirs. L'implémentation Claude Code reste dans `claude-code/`. Les templates dans `core/templates/`. La documentation dans `doc/`. Le `core/` est la partie qui ne change pas entre les providers — c'est le noyau que les repos `voix-mistral`, `voix-gemini` référencent.

## Conséquences

- La structure de `core/` est un contrat — tout changement dans `core/` est un changement de méthode (minor ou major en semver)
- Les contributions peuvent toucher `doc/` sans impacter le cœur
- Pré-requis pour l'i18n (v0.4.0 : `core/fr/` + `core/en/`)
- Gate de qualité : `core/` doit être stable avant publication
