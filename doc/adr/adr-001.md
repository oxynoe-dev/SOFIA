# ADR-001 — Versionnage sémantique (semver)

**Statut** : Proposed
**Date** : 2026-03-30
**Auteur** : Mira

## Contexte

SOFIA est un produit open source publié sur GitHub (oxynoe-dev/sofia). Le versionnage conditionne la communication, la crédibilité auprès des développeurs, et la gestion des breaking changes dans la méthode.

Options considérées :
- **Semver (x.y.z)** — standard GitHub, signal de maturité clair
- **Calver (YYYY.MM)** — adapté aux produits à release continue
- **Pas de versionnage** — juste des commits

## Décision

Adopter semver (x.y.z) pour SOFIA.

- **Major (x)** : changement incompatible dans la structure des artefacts ou le modèle de personas
- **Minor (y)** : nouvel artefact, nouveau guide, enrichissement méthode
- **Patch (z)** : corrections, clarifications, typos

v1.0.0 = première release publique.

## Conséquences

- Chaque release GitHub porte un tag semver
- Le CHANGELOG.md suit le même format que Katen
- Les breaking changes dans les templates ou conventions doivent être documentés dans les release notes
- Le README affiche le badge de version
