# ADR-005 — Marqueur d'instance `sofia.md`

**Statut** : Accepted
**Date** : 2026-04-05
**Auteur** : Mira

## Contexte

SOFIA est une méthode qui s'applique à des projets concrets (instances). Il faut un moyen de savoir si un dépôt est une instance SOFIA, quelle version de la méthode il utilise, et comment le relier au repo sofia/.

Options considérées :
- **Convention implicite** — pas de marqueur, on devine à la structure
- **Fichier de config JSON** — structuré mais illisible sans outil
- **Fichier markdown simple** — lisible par les humains et les LLMs

## Décision

Un fichier `sofia.md` à la racine de chaque instance. Contient : version de la méthode, lien vers le repo voix, date de dernière synchronisation, liste des personas actifs. Format markdown (pas JSON) — lisible par les humains et les LLMs. Le fichier est versionné avec le projet.

## Conséquences

- Chaque instance est auto-descriptive
- La convergence peut scanner les `sofia.md` pour inventorier les instances
- Le template SOFIA fournit un `sofia.md` pré-rempli
- La mise à jour du marqueur est manuelle (pas de sync automatique)
