# ADR-004 — Support multi-plateforme IA

**Statut** : Accepted
**Date** : 2026-03-30
**Auteur** : Mira

## Contexte

SOFIA est conçue et testée sur Claude Code (Anthropic). La méthode repose sur des mécanismes spécifiques : CLAUDE.md, personas via system prompt, isolation par workspace, sessions avec contexte persistant.

D'autres plateformes offrent des capacités comparables ou émergentes :
- **Codex** (OpenAI) — agents de code autonomes
- **Gemini** (Google) — contexte long, multimodal
- **Mistral** (Le Chat) — modèles ouverts, API compatible
- **Cursor, Windsurf, Aider** — IDE augmentés avec rules files

La question : SOFIA reste-t-elle spécifique Claude Code, ou devient-elle une méthode générique d'orchestration multi-agent IA ?

### Options

**A — Claude Code only**
- SOFIA = méthode pour Claude Code, point
- Plus simple, plus profond, plus crédible sur un outil
- Limite l'audience

**B — Noyau commun + guides d'adaptation**
- Extraire les principes génériques (personas, isolation, bus, sessions)
- Un guide par plateforme qui mappe les concepts (CLAUDE.md → .cursorrules, etc.)
- Phase d'analyse nécessaire sur chaque plateforme
- Effort significatif mais portée maximale

**C — Abstraction complète**
- La méthode ne mentionne aucun outil spécifique
- Trop abstrait, perd le concret qui fait la valeur

## Décision

**Option B retenue** — noyau commun dans `sofia` + repos séparés par plateforme.

| Repo | Rôle |
|------|------|
| `oxynoe-dev/sofia` | Noyau méthode + implémentation de référence Claude Code |
| `oxynoe-dev/sofia-mistral` | Adaptation Mistral / Le Chat |
| `oxynoe-dev/sofia-gemini` | Adaptation Gemini (si pertinent, à évaluer) |
| `oxynoe-dev/sofia-codex` | Adaptation Codex / OpenAI (si pertinent, à évaluer) |

Chaque repo d'adaptation :
- Référence `sofia` pour les principes génériques (personas, isolation, bus, sessions)
- Documente le mapping des concepts (CLAUDE.md → équivalent plateforme)
- Contient ses propres templates, exemples, guide de démarrage
- Versionnage indépendant (semver)

**Timeline** : publication v0.3.0 d'abord (avril 2026), feedback utilisateurs, puis v0.5.0 multi-plateforme. Premier test Mistral prévu été 2026. Phase d'analyse avant implémentation. Pas de travail multi-plateforme avant feedback réel sur la v0.3.0.

Points ouverts :
- **Qui** : faut-il un persona dédié ou c'est du travail Mira + test PO ?
- **Noyau extractible** : identifier dans `sofia` ce qui est générique vs ce qui est spécifique Claude Code — pré-requis avant toute adaptation
- **Contribution** : chaque repo d'adaptation peut vivre indépendamment (maintenu par la communauté de la plateforme)

## Conséquences

- `sofia` doit séparer clairement le noyau méthode de l'implémentation Claude Code (structurellement ou par documentation)
- Chaque adaptation demande un test réel, pas un mapping théorique — budget temps à prévoir
- Le naming `sofia-{plateforme}` crée une famille de repos cohérente sous l'org `oxynoe-dev`
