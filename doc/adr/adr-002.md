# ADR-002 — Deux niveaux de pilotage : roadmap produit + backlog assistant

**Statut** : Superseded
**Supersédé par** : ADR-007 (Ownership par roadmap)
**Date** : 2026-03-30
**Auteur** : Mira

## Contexte

L'orchestrateur pilote plusieurs produits avec plusieurs assistants. Deux besoins distincts :
- Savoir où en est chaque **produit** (versions, features, jalons)
- Savoir où en est chaque **assistant** (ses tâches en cours, ses blocages)

L'approche initiale (analyse-voix-convergence.md §6.2) prévoyait un seul fichier `roadmap.md` par workspace avec deux sections (points ouverts + livrables). Le problème : une roadmap produit est **partagée** entre plusieurs assistants (Mira, Nora, Axel travaillent tous sur Katen), alors qu'un backlog est **local** à un assistant.

## Décision

Séparer en deux artefacts :

**Roadmap produit** — `shared/roadmap-{produit}.md`
- Vit dans le bus partagé (`shared/`)
- Pilotée par l'orchestrateur
- Lue par tous les assistants concernés
- Contient : versions, features, jalons, décisions structurantes
- Exemples : `shared/roadmap-katen.md`, `shared/roadmap-sofia.md`, `shared/roadmap-convergence.md`

**Backlog assistant** — `backlog.md` à la racine de chaque workspace
- Local au workspace
- Géré par chaque persona
- Contient : points ouverts avec `source:`, livrables avec `cible:`
- 5 statuts normalisés : done, running, ready, todo, blocked
- Marqueurs `↔` pour les convergences inter-voix

Protocole de session :
- **Ouverture** : scanner `shared/roadmap-*.md` pertinentes + son `backlog.md` + inbox `shared/`
- **Fermeture** : mettre à jour son `backlog.md`

## Conséquences

- Chaque persona crée son propre backlog (pas un tiers qui le fait pour lui)
- Les roadmaps produit ne vivent pas dans les repos produit — elles sont dans le bus partagé
- Convergence parse les deux types : `backlog.md` par workspace + `shared/roadmap-*.md`
- Les templates SOFIA fournissent `backlog.md` et `roadmap-produit.md`

## Note (v0.2.2)

Le système backlog par persona a été supprimé en v0.2.2 (01/04/2026). Les backlogs sont remplacés par des roadmaps produit avec marqueurs @owner. L'arbitrage reste à l'orchestrateur, le suivi passe par les roadmaps dans shared/. Voir ADR-007.
